# Memory RAG Spec

> 本文件承接原 `SKILL.md` 中与资料读取、记忆调用、RAG 检索、来源边界和素材归属有关的规则。
>
> 本阶段只做语义等价迁移：不新增资料来源类型、不改变素材优先级、不改变引用边界、不改变对用户材料和外部材料的区分原则。

---

## 2.0 模块职责

本模块负责规定导演剪辑版写作过程中，如何读取、调用、区分和使用不同来源的材料。

它回答七个问题：

1. 当前任务应优先依据什么材料。
2. 哪些材料可以当作原文依据。
3. 哪些材料只能当作用户理解、外部补充或背景信息。
4. 什么时候需要调用记忆、资料库或 RAG。
5. 检索结果如何进入案例池、机制判断或正文。
6. 不同来源之间发生冲突时如何处理。
7. 如何避免把来源不明的内容写成作者原文。

本模块只负责资料来源、调用边界和使用规则，不负责入口判断、案例搜索、正文写作或最终质量校验。

---

## 2.1 资料来源类型

导演剪辑版可能使用以下几类资料来源。

### 2.1.1 用户直接提供的原文

包括用户粘贴的全文、章节、片段、摘录、扫描识别文本、文件内容或明确说明为原书内容的材料。

这是最优先的依据。

如果用户提供的是原文，应优先基于原文提取主问题、主机制、证据和结构。

不得用外部资料覆盖用户提供的原文判断。

### 2.1.2 用户提供的笔记或理解

包括用户自己的读书笔记、批注、提纲、心得、已有草稿、改写要求、写作方向和个人判断。

这类材料可以用于理解用户目标和读者取向，但不能自动当作作者原文。

如果用户笔记和原文之间存在差异，应明确区分：

```text
原文说了什么
用户理解了什么
本次写作需要如何处理
```

不得把用户笔记伪装成书中观点。

### 2.1.3 目录、封面、简介和公开信息

包括书名、作者、目录、出版信息、封面文案、平台简介、媒体介绍、公开书评、出版社介绍等。

这类材料可以帮助判断主题、范围和可能的问题意识，但不能替代全文阅读。

如果只有这类材料，不得假装已经掌握全书论证。

只能写成基于公开信息的 Quick 版或提出待确认的主问题。

### 2.1.4 记忆与历史材料

包括用户此前保存、上传、整理或在历史任务中形成的资料、案例池、风格偏好、项目规范、迁移记录和已确认规则。

这类材料可以用于保持项目一致性，但不能覆盖用户当前明确指令。

如果历史材料与当前指令冲突，以当前指令为准。

### 2.1.5 RAG 检索结果

包括从资料库、收藏、文档、历史记录、归档内容或相关文件中检索出的片段。

RAG 检索结果只能作为候选材料，必须经过判断后才能使用。

不得因为某段内容被检索出来，就默认它权威、完整、最新或适用于当前任务。

### 2.1.6 外部公开资料

包括公开网页、新闻、报告、论文、评论、百科、数据库和其他可验证来源。

外部公开资料可以用于补充背景、核对事实、寻找现实案例或理解书籍基本信息，但不能替代用户提供的原文。

外部资料必须服务主问题和主机制，不能因为信息量大就进入正文。

### 2.1.7 模型常识

模型常识只能用于一般性语言组织、基础概念解释和常识性连接。

凡涉及具体书中内容、具体事实、具体案例、具体数据、具体年份、具体人物或具体事件，不能只凭模型常识断言。

如果没有可靠来源，应降低确定性表达，或要求用户补充材料。

---

## 2.2 资料优先级

当多个来源同时存在时，按以下优先级处理：

```text
用户当前明确指令
→ 用户当前提供的原文
→ 用户当前提供的目录、摘录、笔记和草稿
→ 项目内已确认规范
→ 用户历史材料或记忆
→ RAG 检索结果
→ 外部公开资料
→ 模型常识
```

优先级的含义是：

1. 高优先级材料决定任务目标和判断边界。
2. 低优先级材料只能补充高优先级材料，不能覆盖它。
3. 如果低优先级材料与高优先级材料冲突，应优先相信高优先级材料。
4. 如果高优先级材料本身不足，应明确说明限制，而不是用低优先级材料填补成"看似完整"的结论。

例如，用户提供了原文片段，就不能用公开书评直接推断全书主旨。

例如，用户明确要求保留原语境，就不能因为历史偏好是中文读者版而自动改成本土化写法。

---

## 2.3 何时调用记忆或 RAG

以下情况应考虑调用记忆或 RAG：

1. 用户提到此前保存、上传、整理过的材料。
2. 用户使用 `@文件名`、`@文件夹名`、`@项目名` 或类似方式引用已有资料。
3. 用户要求延续此前的项目规范、写作风格、迁移进度或案例池。
4. 当前任务需要核对已经拆分出的模块、模板、清单或迁移地图。
5. 用户要求"继续""接着上次""按之前的规则来"。
6. 用户要求从资料库、收藏、归档、历史文档中找内容。
7. 当前上下文不足以判断用户所说对象，但历史材料中可能存在明确指向。

调用记忆或 RAG 的目标不是扩大材料量，而是恢复任务上下文、确认既有规则和避免重复劳动。

---

## 2.4 何时不调用记忆或 RAG

以下情况不应调用记忆或 RAG：

1. 用户当前已经提供了足够完整的材料。
2. 用户明确要求只基于当前文本处理。
3. 任务只是普通翻译、润色、摘要、改写，不涉及历史项目。
4. 调用历史材料可能干扰当前明确要求。
5. 用户要求的是一次性生成，不要求延续前文。
6. 当前问题可以直接根据用户输入完成。
7. 用户要求不要使用历史信息、收藏、记忆或外部资料。

如果用户说"只看这段""不要参考之前""不要联网""不要用旧材料"，必须遵守。

---

## 2.5 检索前的问题转换

调用 RAG 或资料库前，应先把用户需求转换成可检索的问题。

不得直接用抽象词或过宽关键词检索。

应优先检索：

1. 文件名、模块名、章节名。
2. 用户明确提到的术语。
3. 已知迁移目标。
4. 已确认的模板名称。
5. 具体规则名称。
6. 具体案例或案例池名称。
7. 项目中的固定表达。

不推荐检索：

```text
写作
案例
结构
规则
深度
```

推荐检索：

```text
specs/case-policy.md 搜索预算
Step 0 入口判断与模式选择
导演剪辑版 触发识别
案例等级 A B C
validation-checklist 主问题 偏离
```

检索词应服务当前任务，而不是泛泛扩大范围。

---

## 2.6 检索结果使用规则

RAG 检索结果必须经过筛选后使用。

使用检索结果时，应判断：

1. 它是否与当前任务直接相关。
2. 它是原文、用户笔记、项目规范，还是外部补充。
3. 它是否足够完整，还是只是片段。
4. 它是否已经被后续规则更新。
5. 它是否与当前用户指令冲突。
6. 它能否支持当前结论。
7. 它是否只适合作为参考，不适合进入正文。

不得把检索结果机械拼接进输出。

不得因为检索结果里出现某个词，就认定它是当前任务的核心。

不得把多个来源的内容混合后写成同一个来源的观点。

---

## 2.7 来源标注与归属

写作时必须区分材料来源。

至少应能区分：

1. 原书或原文内容。
2. 用户提供的笔记、批注或要求。
3. 项目规范或模板规则。
4. RAG 检索出的历史材料。
5. 外部公开资料。
6. 为帮助理解而生成的解释、类比或转译。

如果内容进入正式正文，应避免来源混淆。

可以使用以下表达：

```text
原文真正要解释的是……
用户当前要求重点处理的是……
根据当前片段可见……
作为中文读者版的补充场景，可以这样理解……
这个案例属于外部补充，不是原书内容……
```

不应使用以下表达：

```text
书中说……
作者认为……
原文证明了……
```

除非确实有原文依据。

## 2.8 冲突处理

不同资料来源发生冲突时，按以下原则处理。

### 2.8.1 当前指令优先

用户当前明确指令优先于历史偏好、旧模板和旧规则。

例如，历史项目默认使用中文读者版，但用户当前要求原语境版，应按原语境版处理。

### 2.8.2 原文优先于外部解读

用户提供的原文优先于书评、简介、百科、平台介绍和外部评论。

外部解读可以帮助理解背景，但不能替代原文判断。

### 2.8.3 已确认版本优先于默认版本

如果用户已经确认 Quick、Standard、Deep 或中文读者版、原语境版、对照版，应按确认版本执行。

不得在后续写作中自动切换模式。

### 2.8.4 新规则优先于旧规则

如果同一项目中存在新旧两版规范，应优先使用用户明确要求的最新版本。

如果无法判断新旧，应向用户确认，不应自行合并。

### 2.8.5 不确定时标注限制

如果冲突无法解决，应明确说明限制并请求确认。

不得用模糊措辞掩盖来源冲突。

---

## 2.9 素材不足时的处理

当资料不足时，不得伪造完整性。

常见不足包括：

1. 只有书名，没有正文。
2. 只有目录，没有论证。
3. 只有片段，却要求全书判断。
4. 只有用户笔记，缺少原文。
5. 只有外部简介，缺少作者论证。
6. 检索结果只有零散片段，不能支撑结构判断。
7. 材料中缺少版权主体、作者身份、文本归属或版本信息。

处理方式：

1. 明确说明"当前只能基于已有材料判断"。
2. 降级为 Quick 或局部版。
3. 请求用户补充目录、摘录、章节或全文。
4. 将判断写成"可能""初步""基于当前片段"。
5. 不生成伪装完整的全书结构。
6. 不编造章节、页码、论证顺序或作者立场。

如果版权主体、作者身份或文本归属不清，应先确认素材边界，再决定是否进入改写、引用或发布级成稿。

---

## 2.10 记忆材料的使用边界

记忆材料用于保持连续性，不用于替用户做未经确认的决定。

可以使用记忆材料来：

1. 延续项目进度。
2. 恢复已确认的文件结构。
3. 复用已确认的术语。
4. 保持模板和风格一致。
5. 避免重复迁移同一内容。
6. 找回用户此前确认过的边界条件。

不应使用记忆材料来：

1. 推翻用户当前指令。
2. 自动暴露用户过去的私人内容。
3. 把旧项目规则套到新项目。
4. 在用户没有要求时引入无关历史材料。
5. 将历史偏好当成永久偏好。
6. 把旧案例当成当前事实。

记忆只提供上下文，不提供最终判断权。

---

## 2.11 外部资料使用边界

外部资料只能作为补充，不得替代原文和用户当前材料。

适合使用外部资料的情况：

1. 核对书籍基本信息。
2. 查找公开出版信息。
3. 理解必要背景。
4. 寻找现实案例。
5. 验证具体事实、年份、数据或事件。
6. 补足中文读者理解所需的公共背景。

不适合使用外部资料的情况：

1. 用书评替代读原文。
2. 用百科替代机制提取。
3. 用热点替代作者问题意识。
4. 用他人解读覆盖用户提供的文本。
5. 用不可靠来源支撑关键判断。
6. 为了显得资料丰富而加入无关信息。

外部资料进入正文时，应服务主问题和主机制，并符合案例政策与证据剪裁规则。

---

## 2.12 RAG 输出前检查

使用记忆或 RAG 后，输出前应检查：

1. 是否把当前用户指令放在最高优先级。
2. 是否区分了原文、用户笔记、项目规范和外部补充。
3. 是否误把检索片段当成完整原文。
4. 是否误把用户笔记当成作者观点。
5. 是否误把外部资料当成原书内容。
6. 是否存在新旧规则冲突。
7. 是否需要向用户确认版本、素材或来源。
8. 是否在素材不足时做了限制说明。
9. 是否避免了无关历史材料干扰当前任务。
10. 是否遵守案例政策、写作政策和质量校验清单。

如果检查未通过，应先修正来源边界，再继续写作。

---

## 2.13 与其他模块的关系

本模块为其他模块提供资料边界。

具体关系如下：

1. 入口判断依赖本模块判断素材类型和素材完整度。
2. 主问题与机制提取依赖本模块区分原文、笔记和外部补充。
3. 案例政策依赖本模块判断案例来源、事实可靠性和使用边界。
4. 骨架模板依赖本模块提供可用材料范围。
5. 正文写作依赖本模块避免来源混淆。
6. 输出政策依赖本模块决定是否需要标注限制、来源和版本说明。
7. 质量校验清单依赖本模块检查事实、来源、引用和归属问题。

如果资料边界不清，不应直接进入完整成稿。
