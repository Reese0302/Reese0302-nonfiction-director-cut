## 1.0 模块职责

本模块负责在进入案例搜索、骨架生成和正文写作之前，完成导演剪辑版的核心判断：

1. 这篇内容真正要回答什么问题。
2. 原文最重要的解释机制是什么。
3. 哪些内容只是信息、例子或背景，不能误当成主线。
4. 应该如何避免把导演剪辑版写成摘要、读书笔记或资料汇编。
5. 后续案例、结构和正文应围绕什么中心展开。

本模块只负责主问题与机制提取，不负责案例搜索、正文成稿、标题包装或质量校验。

---

## 1.1 导演剪辑版不是摘要

导演剪辑版的目标不是把原文压缩得更短，而是把原文背后的问题意识、解释机制和读者理解路径显影出来。

普通摘要回答的是：

```text
这篇文章 / 这本书讲了什么？
```

导演剪辑版要回答的是：

```text
它真正想解释什么问题？
它为什么这样解释？
这个解释对读者理解现实有什么帮助？
```

因此，不能只按章节顺序复述内容，也不能把原文观点排列成要点清单。

如果输出只是"第一章说了什么、第二章说了什么、作者认为一二三"，即使信息准确，也不合格。

导演剪辑版必须把材料重组为一条解释链：

```text
问题 → 机制 → 证据 → 现实场景 → 读者理解变化
```

其中，"问题"和"机制"是主线；案例、金句、背景和资料都只能服务这条主线。

---

## 1.2 主问题提取

进入写作前，必须先提取主问题。

主问题不是书名，不是主题词，也不是领域标签。

例如：

```text
教育
消费
亲密关系
组织管理
现代社会
技术变化
```

这些都只是主题，不是主问题。

合格的主问题应当具有解释张力，通常表现为：

1. 一个常见现象为什么会发生。
2. 一个看似矛盾的结果为什么成立。
3. 一个被误解的问题应该如何重新理解。
4. 一个抽象概念如何影响普通人的具体生活。
5. 一个时代变化如何改变人的选择、关系或行动方式。

主问题应该尽量写成一句可回答的问题，而不是一个名词短语。

示例：

```text
不合格：内卷。
合格：为什么很多人明明越来越努力，却越来越感觉不到进步？
```

```text
不合格：消费主义。
合格：为什么人们知道自己不需要，却还是会不断购买？
```

```text
不合格：亲密关系。
合格：为什么越想维持稳定关系的人，反而越容易在关系里感到疲惫？
```

主问题一旦确定，后续所有内容都必须围绕它展开。

如果某个观点、案例或段落不能帮助回答主问题，即使它本身精彩，也不应进入正文主线。

---

## 1.3 主问题提取方法

提取主问题时，应优先从以下位置寻找：

1. 原文反复出现的核心困惑。
2. 作者最想反驳的常识判断。
3. 原文中解释篇幅最长的矛盾或现象。
4. 多个章节共同指向的深层问题。
5. 用户要求中特别强调的读者痛点。
6. 能把原文内容连接成一条线的问题。

不得只根据标题、目录或单个高频词确定主问题。

如果材料是全文型，应综合全文判断。

如果材料是片段型，只能提取"当前片段的主问题"，不得冒充全书主问题。

如果材料是书名型，除非有足够公开资料支撑，否则只能提出"可能的主问题"，并标注来源限制。

---

## 1.4 主问题筛选标准

一个合格的主问题应同时满足以下标准：

1. 能覆盖原文最重要的论证内容。
2. 能让读者产生"这确实是我也关心的问题"的感觉。
3. 能引出机制解释，而不是只能引出观点罗列。
4. 能连接原书语境与目标读者语境。
5. 能排除无关材料，形成取舍标准。
6. 能支撑后续案例搜索和正文结构。

如果存在多个候选主问题，应优先选择最能统摄全文、最能改变读者理解、最能支撑机制展开的那个。

不应选择过大的问题。

例如：

```text
人类为什么痛苦？
社会为什么变化？
我们应该如何生活？
```

这类问题太大，容易导致正文空泛。

也不应选择过小的问题。

例如：

```text
作者在第三章为什么举了某个例子？
某个术语是什么意思？
某个段落怎样改写？
```

这类问题只能支撑局部解释，不能支撑完整导演剪辑版。

---

## 1.5 机制识别

主问题确定后，必须识别原文用来回答这个问题的主机制。

机制不是观点，也不是结论。

观点通常是：

```text
作者认为 X 很重要。
作者批评了 Y。
作者提出应该 Z。
```

机制回答的是：

```text
为什么 X 会发生？
X 是怎样一步步形成的？
哪些条件让 X 变得稳定、普遍或难以改变？
```

合格的机制通常具有因果链条。

示例：

```text
不合格：人们很焦虑。
合格：评价标准单一化 → 比较对象扩大 → 努力回报变得不确定 → 焦虑常态化。
```

```text
不合格：平台影响生活。
合格：平台降低选择成本 → 增加即时反馈 → 改变用户注意力分配 → 重塑消费和社交习惯。
```

```text
不合格：关系需要沟通。
合格：期待没有被说出 → 对方按旧模式回应 → 失望积累 → 沟通变成指责。
```

机制必须能够解释现象，而不是只表达态度。

---

## 1.6 主机制与辅助机制

导演剪辑版必须区分主机制和辅助机制。

主机制是回答主问题的核心解释链，正文应围绕它展开。

辅助机制是帮助解释主机制某个环节的局部原因、背景条件或补充说明。

不得把所有机制平铺成并列清单。

如果所有机制都同等重要，正文会失去中心，读者也无法判断真正的解释重点。

处理原则：

1. 每篇文章只能有一个最核心的主问题。
2. 每个主问题应有一条最重要的主机制。
3. 辅助机制只能服务主机制，不能另起炉灶。
4. 如果辅助机制过多，应合并、降级或放入案例池 / 备选材料。
5. 正文结构应体现主次关系，而不是把所有观点平均分配篇幅。

---

## 1.7 三层解释

导演剪辑版的解释应尽量形成三层结构：

```text
第一层：读者已经看见的现象。
第二层：现象背后的机制。
第三层：机制如何改变读者对自身处境或现实问题的理解。
```

第一层负责让读者进入问题。

它通常对应生活场景、社会现象、原书中的关键画面或读者熟悉的经验。

第二层负责解释为什么。

它必须讲清楚现象背后的因果链条、结构条件、心理机制、制度安排、文化背景或行动逻辑。

第三层负责改变理解。

它要回答：读者读完后，对这个问题的看法和原来有什么不同？

如果只有第一层，文章会变成故事或案例堆砌。

如果只有第二层，文章会变成抽象理论说明。

如果没有第三层，文章虽然解释了现象，但没有完成导演剪辑版最重要的理解转换。

---

## 1.8 避免摘要化

写作中必须避免以下摘要化倾向：

1. 按章节顺序复述。
2. 罗列作者观点。
3. 用大量"作者认为""书中提到""这一章讲了"推进正文。
4. 只解释概念，不解释概念为什么重要。
5. 只列举案例，不说明案例如何推动机制理解。
6. 把原文结构原样搬进正文，而不重建读者理解路径。
7. 用金句替代分析。
8. 用结论替代机制。

判断一段文字是否摘要化，可以问：

```text
这段是在复述原文，还是在帮助读者理解问题为什么成立？
```

如果答案只是复述，应改写为机制解释。

---

## 1.9 避免过度发挥

导演剪辑版需要重组和转译，但不能脱离原文任意发挥。

以下做法不允许：

1. 把原文没有的立场写成作者立场。
2. 把用户笔记误写成原书内容。
3. 把外部案例当成原文证据。
4. 用现实热点覆盖原文问题。
5. 为了让文章更有冲突感而夸大原文判断。
6. 为了适配中文读者而改变原文核心意思。
7. 在素材不足时假装已经掌握全书结构。

导演剪辑版可以解释、转译、重组，但必须保持原文问题意识和核心机制不被扭曲。

如果需要补充现实案例，必须明确案例是"外部补充"，不能伪装成原书内容。

---

## 1.10 机制输出格式

完成本模块后，应形成一个简短的机制判断结果，用于指导后续模块。

机制判断结果至少包含：

1. 主问题。
2. 主机制。
3. 辅助机制。
4. 原文关键证据。
5. 读者理解难点。
6. 需要补足的现实场景或案例方向。
7. 不应展开的偏题内容。

内部可使用如下格式：

```text
主问题：
这篇内容真正要回答的是……

主机制：
A → B → C → D

辅助机制：
1. ……
2. ……

读者理解难点：
读者可能会误以为……，但原文真正要解释的是……

案例方向：
需要寻找能说明……的现实场景，而不是泛泛寻找……类热点。

取舍边界：
不展开……，因为它不能服务主问题。
```

这个机制判断结果可以不完整展示给用户，但后续写作必须遵守。

---

## 1.11 与后续模块的关系

本模块完成后，后续模块按以下方式调用结果：

1. 案例政策根据主机制决定搜索方向和案例等级。
2. 骨架模板根据主问题和主机制安排文章结构。
3. 正文写作围绕主机制展开，不按原文顺序机械复述。
4. 质量校验清单检查文章是否偏离主问题、是否摘要化、是否案例堆砌。
5. 如果后续发现主问题或主机制判断错误，应回到本模块重新判断，而不是继续硬写。

主问题和主机制一旦确定，就是整篇导演剪辑版的中轴线。

## 4.0 模块职责

本模块负责规定导演剪辑版进入正文写作后的表达原则、段落组织、解释方式和成稿边界。

它回答五个问题：

1. 正文应如何围绕主问题展开。
2. 如何把机制写成读者能理解的叙述。
3. 如何使用原文证据、现实案例和现象类比。
4. 如何避免写成摘要、评论、鸡汤、资料汇编或热点拼贴。
5. 如何在忠实原文与中文读者转换之间保持平衡。

本模块只负责正文写作与表达规则，不负责入口判断、案例搜索、骨架模板或最终质量校验。

---

## 4.1 正文主线

导演剪辑版正文必须围绕主问题和主机制展开。

正文不是按照原书章节顺序机械复述，也不是把案例、观点、金句和资料依次堆上去。

正文应形成一条清晰的理解路径：

```text
读者熟悉的困惑或现象
→ 原文真正要解释的问题
→ 机制如何发生
→ 原文证据如何支撑机制
→ 现实案例或中文场景如何帮助理解
→ 读者应如何重新理解这个问题
```

每一节、每一段、每一个案例都应服务这条路径。

如果某个内容不能推动读者理解主机制，即使它本身有趣、热门或信息量大，也不应进入正文主线。

---

## 4.2 开头写法

开头的任务不是概括全书，而是把读者带入主问题。

合格的开头应满足：

1. 能让读者意识到"这个问题和我有关"。
2. 能提出一个有解释张力的问题。
3. 能自然引出原文的核心价值。
4. 不直接堆概念、作者履历、章节介绍或抽象判断。
5. 不用夸张标题党制造虚假冲突。

开头可以从以下方式进入：

1. 一个中文读者熟悉的生活场景。
2. 一个常见但被误解的社会现象。
3. 一个原文中的关键画面。
4. 一个反常识的问题。
5. 一个读者以为自己懂、但其实没有真正解释清楚的困惑。

不合格开头示例：

```text
本书主要讲了三个方面：第一……第二……第三……
```

```text
作者在第一章提出……在第二章分析……在第三章总结……
```

合格开头应更接近：

```text
很多人以为，问题出在自己不够努力。但这本书真正提醒我们的是：当评价规则改变以后，努力本身也会被重新定义。
```

开头可以简短，但必须建立问题意识。

---

## 4.3 概念解释规则

正文可以解释概念，但不能把概念解释当成正文主体。

解释概念时，应遵守：

1. 先说明它解决什么问题，再说明它是什么意思。
2. 先放进场景，再给出定义。
3. 先讲读者为什么会误解，再讲原文如何重新定义。
4. 尽量少用抽象名词套抽象名词。
5. 不用术语密度制造深度感。

不推荐：

```text
所谓 X，是指在 Y 条件下，由 Z 结构产生的复合型状态。
```

推荐：

```text
你可以把 X 理解为一种处境：人以为自己是在自由选择，其实选择范围早已被平台、规则和他人的期待提前设定好了。
```

概念解释必须回到机制：

```text
这个概念重要，不是因为它听起来高级，而是因为它解释了为什么……
```

如果一个概念只适合做背景说明，不应占据正文核心篇幅。

---

## 4.4 机制写法

机制写作必须把因果链条写清楚。

不能只写结论：

```text
所以人们会焦虑。
所以关系会变差。
所以消费会增加。
```

必须写清楚中间过程：

```text
评价标准变少以后，人们更容易把自己放进同一条赛道比较；比较对象扩大以后，个人努力带来的确定感下降；当努力不能稳定换来结果时，焦虑就不再是偶发情绪，而会变成一种日常背景。
```

机制写作应优先使用：

1. 因果连接。
2. 过程展开。
3. 条件说明。
4. 反常识解释。
5. 从读者经验到抽象机制的过渡。

机制写作应避免：

1. 只给判断不给过程。
2. 只讲态度不讲原因。
3. 只引用原文不解释含义。
4. 只堆案例不抽象机制。
5. 把复杂机制写成单一原因。

如果机制较复杂，可以分层解释，但每一层都要回到主问题。

---

## 4.5 原文证据使用规则

原文证据用于支撑机制，不用于装饰正文。

可以使用的原文证据包括：

1. 原文关键论断。
2. 原文核心案例。
3. 原文反复出现的概念。
4. 原文中的典型场景。
5. 原文的论证转折。
6. 原文对常识判断的反驳。

使用原文证据时，应遵守：

1. 不大段搬运原文。
2. 不把引用当作分析。
3. 不用零散金句替代论证。
4. 不脱离上下文截取原文。
5. 不改变原文立场。
6. 不把用户笔记写成原文证据。
7. 不把外部案例写成原文内容。

每个原文证据后，必须回答：

```text
这个证据说明了机制中的哪一环？
它如何帮助读者理解主问题？
```

如果回答不了，就说明这个证据不应进入正文主线。

---

## 4.6 现实案例与中文场景写法

现实案例和中文场景的任务，是帮助目标读者理解机制，而不是替代原文。

使用现实案例时，应遵守案例政策中的案例等级、本土化原则、案例禁区、搜索预算和证据剪裁规则。

正文中写现实案例时，应做到：

1. 先说明案例对应机制中的哪一环。
2. 再描述必要事实或现象。
3. 最后回到机制解释。
4. 不为了热闹堆热点。
5. 不让案例抢走原文主线。
6. 不消费个体苦难。
7. 不把事实不清的事件写成确定结论。

现实案例不能只证明：

```text
现实中也有这种现象。
```

它必须进一步说明：

```text
这个现象为什么会发生？
它和原文机制之间是什么关系？
它帮助中文读者补上了哪一层理解？
```

如果现实材料只能增加热闹感，不能推动理解，应放入案例池，不进入正文。

---

## 4.7 段落组织

正文段落应遵循"单段单任务"。

一个段落只承担一个主要功能：

1. 提出问题。
2. 描述现象。
3. 解释机制。
4. 引入证据。
5. 分析案例。
6. 完成转折。
7. 推进读者理解。

不要在同一段里同时完成太多任务。

推荐段落结构：

```text
第一句：提出本段要解释的判断。
中间：展开机制、证据或案例。
最后：回到主问题，完成推进。
```

段落之间要有明确推进关系。

不应出现大量并列段落：

```text
此外……
同时……
另外……
还有……
再者……
```

如果正文长期依靠这些词推进，说明结构可能已经变成观点堆砌，而不是机制展开。

---

## 4.8 过渡与转折

导演剪辑版需要用过渡帮助读者从现象进入机制，从原文进入现实，从理解进入判断。

好的过渡不是形式连接，而是理解推进。

常见过渡任务包括：

1. 从表面现象转入深层机制。
2. 从原书语境转入中文读者语境。
3. 从一个案例转回主机制。
4. 从作者观点转入读者处境。
5. 从误解转入重新理解。

可使用的过渡方式：

```text
真正关键的不是……而是……
这件事表面上看是……，但原文要解释的是……
放到中文读者熟悉的场景里，它更像是……
这个案例重要，不是因为它特殊，而是因为它把机制中的一环放大了。
```

避免空泛过渡：

```text
由此可见……
综上所述……
总的来说……
这给我们很多启示……
```

这些表达可以使用，但不能替代具体分析。

---

## 4.9 语言风格

导演剪辑版应使用清晰、具体、可读的语言。

整体风格应满足：

1. 有解释力，但不故作高深。
2. 有现实感，但不迎合情绪。
3. 有判断力，但不替原文强行表态。
4. 有文学性，但不牺牲清楚。
5. 有中文表达习惯，但不改变原文核心意思。

应避免：

1. 学术腔堆叠。
2. 互联网情绪化表达。
3. 过度口号化。
4. 鸡汤式升华。
5. 伪深刻金句。
6. 只靠排比制造气势。
7. 用"我们都知道""显而易见"跳过论证。

好的语言应让读者觉得：

```text
我原来隐约感到这个问题存在，现在终于知道它是怎么发生的。
```

而不是只觉得：

```text
这段话很漂亮，但不知道解释了什么。
```

---

## 4.10 忠实与转译

导演剪辑版可以重组、解释和转译原文，但不能扭曲原文。

忠实不是逐句复述，而是保持原文的问题意识、核心机制和判断边界。

转译不是任意本土化，而是把原文机制转换成目标读者能够理解的经验和语言。

必须区分：

1. 原文明确说了什么。
2. 根据原文机制可以合理推出什么。
3. 为帮助中文读者理解而补充了什么。
4. 作者没有说、不能写成作者立场的内容。

如果正文加入外部案例，应明确它是现实补充。

如果正文加入中文场景，应明确它是语境转换。

如果正文加入作者未直接表达的推论，应保持谨慎，不能写成确定的原文结论。

---

## 4.11 结尾写法

结尾的任务不是喊口号，也不是简单复述全文，而是完成理解转换。

结尾应回答：

```text
读者读完这篇导演剪辑版之后，对主问题的理解发生了什么变化？
```

合格结尾可以完成以下任务：

1. 回到开头提出的问题。
2. 压缩主机制。
3. 指出原文真正改变的理解方式。
4. 说明这个机制对读者理解现实的意义。
5. 保持开放，但不空泛升华。

不推荐结尾：

```text
所以我们要保持清醒，努力生活，成为更好的自己。
```

```text
总之，这本书非常值得一读，能给我们很多启发。
```

推荐结尾更接近：

```text
所以，这本书真正改变的不是我们对某个现象的态度，而是我们对"原因"的理解。很多时候，问题不是个人突然变得脆弱、懒惰或不理性，而是他所处的规则已经改变了行动的回报、关系的结构和自我解释的方式。
```

结尾不必过长，但必须让主问题完成闭环。

## 4.12 禁止项

正文写作中禁止以下做法：

1. 把导演剪辑版写成章节摘要。
2. 把导演剪辑版写成读后感。
3. 把导演剪辑版写成资料汇编。
4. 把导演剪辑版写成热点评论。
5. 把导演剪辑版写成概念百科。
6. 把导演剪辑版写成金句合集。
7. 把导演剪辑版写成案例堆砌。
8. 把现实案例写成原书内容。
9. 把用户提供的笔记写成作者原文。
10. 在素材不足时假装已经掌握全文。
11. 为了中文读者转换而改变原文核心机制。
12. 为了增强冲突而夸大作者立场。
13. 为了追求完整而塞入不服务主问题的内容。

如果出现以上倾向，应回到主问题和主机制重新裁剪。

---

## 4.13 正文生成前检查

进入正文成稿前，应至少确认：

1. 主问题已经明确。
2. 主机制已经明确。
3. 正文骨架不是章节复述。
4. 案例数量符合案例政策。
5. 每个案例都能服务机制。
6. 原文证据与外部补充已经区分。
7. 中文读者转换没有改变原文核心意思。
8. 结尾准备完成理解转换，而不是口号升华。

如果以上条件不满足，不应直接生成完整正文。

## 4.14 与其他模块的关系

本模块依赖入口判断、主问题与机制提取、案例政策和骨架模板。

具体关系如下：

1. 入口判断决定正文采用中文读者版、原语境版还是对照版。
2. 主问题与机制提取决定正文主线。
3. 案例政策决定现实案例是否可用、如何搜索、如何剪裁。
4. 骨架模板决定正文结构的基础形态。
5. 质量校验清单用于检查正文是否符合本模块规则。

如果正文写作中发现主问题不成立，应回到 Step 1 重新判断。

如果正文写作中发现案例不合适，应回到案例政策重新筛选。

如果正文结构失控，应回到骨架模板重新组织。
