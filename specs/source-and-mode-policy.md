# Source and Mode Policy

> 本文件承接原 `SKILL.md` 中 `Step 0` 的入口判断逻辑。
>
> 本阶段只做语义等价迁移：不新增入口类型、不改变模式判断标准、不改变版本确认规则。

---

## 0.0 模块职责

本模块负责在进入正式写作流程前，判断用户输入属于哪一种任务入口，并决定后续应进入哪种处理模式。

它回答五个问题：

1. 用户是不是在触发导演剪辑版？
2. 用户给的是书名、全文、片段，还是混合素材？
3. 用户要求的是中文读者版、原语境版，还是对照版？
4. 本次应使用 Quick、Standard，还是 Deep？
5. 是否需要先向用户确认版本、素材或模式？

本模块只负责入口判断，不负责正文写作、案例搜索、骨架生成或质量校验。

---

## 0.1 触发识别

当用户输入中出现以下意图时，应识别为导演剪辑版任务：

1. 要求"导演剪辑版"。
2. 要求对一本书、一篇文章、一个概念、一段材料做"展开版""深读版""拆解版""中文读者版"。
3. 要求把内容改写为更适合中文读者理解的长文。
4. 要求补充现实案例、中文案例、生活场景、社会现象。
5. 要求从读者能理解的角度重写，而不是仅做摘要。
6. 用户明确提到本 skill、导演剪辑、案例池、三层解释等内部术语。

如果用户只是要求普通摘要、翻译、润色、提纲、标题、短评，不自动进入导演剪辑版，除非用户明确要求使用本 skill。

## 0.2 素材类型判断

进入导演剪辑版后，必须先判断用户提供的素材类型。

### 0.2.1 书名型

用户只提供书名、作者名、链接、ISBN、封面或一句"帮我写某书导演剪辑版"。

书名型素材默认信息不足。不能假装已经读过全文。

处理原则：

1. 如果无法访问全文，只能基于公开信息、用户补充内容和可验证资料工作。
2. 必须标注素材来源限制。
3. 若用户要求高质量输出，应建议用户提供目录、章节、摘录或全文。
4. 不得虚构书中细节、页码、章节结构或作者论证。

### 0.2.2 全文型

用户提供完整文章、完整章节、完整书稿，或足够长、足以支撑结构分析的材料。

全文型可进入完整处理流程。

处理原则：

1. 优先依据用户提供文本。
2. 可以抽取核心问题、主机制、章节结构、关键证据。
3. 现实案例必须服务原文机制，而不是替代原文。
4. 不得用外部资料覆盖原文判断。

### 0.2.3 片段型

用户只提供一段摘录、几页内容、部分章节、零散笔记或局部观点。

片段型可以写局部导演剪辑版，但不能冒充全书判断。

处理原则：

1. 必须说明"基于当前片段"。
2. 只分析片段中可见的问题、机制和证据。
3. 不推断全书结构。
4. 如需扩展为全书版，应要求用户补充目录或更多文本。

### 0.2.4 混合型

用户同时提供书名、目录、摘录、笔记、链接、要求、已有草稿等多种素材。

混合型应优先使用用户直接提供的文本，其次使用目录和笔记，再其次使用外部可验证资料。

处理原则：

1. 建立素材优先级。
2. 区分"原文内容""用户理解""外部补充"。
3. 对不确定内容做标注。
4. 不把用户笔记误当成作者原文。

## 0.3 用户视角判断

进入导演剪辑版前，必须判断用户希望文章站在哪个读者语境中。

### 0.3.1 中文读者版

默认模式。

适用于用户要求：

1. 给中文读者看。
2. 加中文案例。
3. 接地气。
4. 结合中国语境。
5. 让普通读者看懂。
6. 改写成公众号、长文、视频稿、课程稿。

中文读者版必须优先考虑中文读者的经验、制度环境、社会语感、生活场景和表达习惯。

### 0.3.2 原语境版

适用于用户明确要求保留原书语境、原作者问题意识、原国家或原文化背景。

原语境版可以少量解释中文读者不熟悉的制度和文化背景，但不能强行替换为中文案例。

### 0.3.3 对照版

适用于用户要求中外对照、原语境与中文语境对比、两个版本并列。

对照版必须明确区分：

1. 原书语境中，这个问题为什么成立。
2. 中文语境中，读者会如何理解。
3. 哪些机制相同。
4. 哪些案例不能直接平移。
5. 哪些概念需要重新翻译。

## 0.4 模式判断

进入导演剪辑版后，必须判断本次任务使用哪种工作模式。

模式不是按文章长短随意决定，而是根据素材完整度、用户目标、输出深度和搜索需求共同判断。

### 0.4.1 Quick 模式

Quick 模式适用于快速生成、局部改写、轻量展开或用户没有要求深度研究的场景。

适用情况：

1. 用户只要求快速看懂。
2. 用户只要求一篇短文、简版、口播稿、公众号初稿。
3. 用户提供的素材较少，无法支撑完整深读。
4. 用户只需要局部片段的解释或改写。
5. 用户明确要求"先来一版""快速版""不用太长"。

Quick 模式的处理原则：

1. 不强制搜索。
2. 不展开复杂案例池。
3. 优先保证主问题清楚、机制清楚、表达顺畅。
4. 可以使用常见生活现象类比，但必须标注案例等级。
5. 不得伪装成完整深读版。

### 0.4.2 Standard 模式

Standard 模式是默认模式。

适用情况：

1. 用户要求完整导演剪辑版，但没有要求极深研究。
2. 用户提供了较完整素材，足以支撑结构分析。
3. 用户希望文章有现实案例、有中文读者转换、有机制解释。
4. 用户目标是生成一篇可读、完整、可发布的长文。
5. 用户没有明确指定 Quick 或 Deep。

Standard 模式的处理原则：

1. 必须识别主问题。
2. 必须建立主机制。
3. 必须有结构骨架。
4. 必须建立案例池，但不堆案例。
5. 搜索预算按案例政策执行。
6. 输出应兼顾可读性、解释力和原文忠实度。

### 0.4.3 Deep 模式

Deep 模式适用于用户明确要求深度研究、长篇拆解、课程级内容或高质量成稿的场景。

适用情况：

1. 用户明确要求 Deep、深度版、研究版、课程稿、长篇专题。
2. 用户提供全文、完整章节、完整书稿或大量材料。
3. 用户要求多层机制分析。
4. 用户要求大量现实案例、跨语境对照或系统性重写。
5. 用户希望产出接近最终发布稿或课程讲义。

Deep 模式的处理原则：

1. 必须完整执行入口判断、主问题提取、机制拆解、案例池建设和质量校验。
2. 搜索预算按案例政策执行。
3. 必须控制证据数量，不能把深度理解误解为资料堆砌。
4. 必须保留原文问题意识，同时完成中文读者转换。
5. 如果素材不足以支撑 Deep，应先要求用户补充材料，或降级为 Standard / Quick。

### 0.4.4 模式优先级

当用户没有明确指定模式时，按以下顺序判断：

1. 如果素材不足且用户只要求快速理解，使用 Quick。
2. 如果素材足够且用户要求完整文章，使用 Standard。
3. 如果用户明确要求深度研究，且素材足够，使用 Deep。
4. 如果用户明确要求 Deep 但素材不足，必须先说明限制，并询问是否补充素材或降级。
5. 如果用户明确要求 Quick，即使素材很充分，也不主动升级为 Deep。

不得因为模型可以多写，就自动把 Quick 升级为 Deep。

## 0.5 版本确认

进入正文写作前，必须判断是否需要向用户确认版本。

### 0.5.1 必须确认的情况

以下情况必须先确认，不得直接进入正文：

1. 用户只给书名，但要求完整导演剪辑版。
2. 用户要求 Deep，但素材不足。
3. 用户没有说明希望中文读者版、原语境版还是对照版，且任务明显依赖语境转换。
4. 用户提供的是片段，却要求全书判断。
5. 用户要求加入现实案例，但案例方向可能影响文章立场或读者理解。
6. 用户同时提出多个互相冲突的目标，例如"极短"又"极深"、"只摘要"又"导演剪辑版"。
7. 用户提供的素材来源、版权主体、作者身份或文本归属不清，可能影响引用和改写边界。

### 0.5.2 不必确认的情况

以下情况可以直接进入处理：

1. 用户明确指定导演剪辑版、中文读者版和输出形式。
2. 用户提供了足够素材，且目标清楚。
3. 用户明确要求"先按默认来"。
4. 用户要求 Quick 模式的局部解释或改写。
5. 用户只要求根据当前片段写一版，不要求全书判断。

不必确认时，也应在输出开头简要标明本次采用的默认判断。

### 0.5.3 默认版本

如果用户没有特别说明，默认采用：

```text
中文读者版 + Standard 模式
```

但默认成立需要满足两个条件：

1. 用户触发了导演剪辑版或等价任务。
2. 用户提供的素材足以支撑 Standard 模式。

如果素材不足，则默认不能成立，必须降级或确认。

### 0.5.4 确认方式

确认问题必须简短、具体、可选择。

优先使用三选一或二选一，不要让用户重新描述全部需求。

示例：

```text
我先确认一下：这次按哪种版本写？

A. 中文读者版：重点做中文语境转换和现实案例
B. 原语境版：尽量保留原书语境
C. 对照版：原语境和中文语境并列比较
```

如果问题是素材不足，确认方式应直接指出限制：

```text
目前只有书名，不能当作已经读过全文。你希望：

A. 先基于公开信息写 Quick 版
B. 你补充目录/摘录后，我再写 Standard 版
C. 你提供全文后，我再写 Deep 版
```

## 0.6 入口输出

完成入口判断后，应形成一个简短的入口判断结果，用于指导后续模块。

入口输出不一定要展示给用户，但内部必须明确。

### 0.6.1 入口输出字段

入口判断至少包含：

1. 任务类型：是否为导演剪辑版。
2. 素材类型：书名型、全文型、片段型或混合型。
3. 用户视角：中文读者版、原语境版或对照版。
4. 工作模式：Quick、Standard 或 Deep。
5. 是否需要确认：是或否。
6. 限制说明：素材不足、语境不明、案例方向不明等。
7. 下一步动作：进入主问题提取、请求补充素材、请求版本确认或降级处理。

### 0.6.2 面向用户的简短说明

如果直接进入写作，应在必要时用一句话说明默认判断。

示例：

```text
我会按"中文读者版 + Standard 模式"处理：优先忠实原文机制，同时补足中文读者能理解的场景和案例。
```

如果不能直接进入写作，应先提出确认问题，不输出正文。

### 0.6.3 与其他模块的关系

本模块完成后，后续模块按入口结果调用：

1. 若任务不是导演剪辑版，按普通摘要、翻译、润色或问答处理。
2. 若任务是导演剪辑版，进入主问题与机制提取。
3. 若需要案例，调用案例政策。
4. 若需要模板，调用骨架模板。
5. 若进入最终成稿，调用质量校验清单。

入口判断不得替代后续模块，也不得提前生成正文结论。
