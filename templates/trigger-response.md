# Trigger Response Template

来源：`SKILL.md` → `Step 0.1 仅触发词输入时的固定回应`

迁移状态：已从 monolith baseline 迁移。

## 用途

当用户只输入触发词，且没有同时提供书名、文件、正文或链接时，使用本模板回应。

本模板只负责引导用户提供素材和选择模式，不直接进入正文生成。

## 触发条件

当用户只输入以下任一触发词，且没有同时提供书名、文件或书籍内容时：

- `/director-cut`
- `/book-localize`
- `/非虚构导演剪辑版`
- `/中文导演剪辑版`
- `/导演剪辑版`

不要直接开始生成。必须先向用户介绍本 Skill 的三种模式，并请用户提供书籍材料。

## 固定回应模板

```markdown
这是「非虚构内容中文导演剪辑版」Skill。

它不是逐章/逐段总结，也不是简单替换案例，而是把一份非虚构内容重新剪成一条中文读者能顺着读下去的问题链：先识别或提炼主问题，再重组核心机制，最后用合适的案例帮助理解。

支持四种素材类型：

| 素材类型 | 适合场景 |
|---|---|
| 非虚构书 | 一本书的解读，包括纸质书、电子书 |
| 长文章 | 公众号长文、Substack、X 长推文、播客文字稿等 |
| 系列内容 | 同一作者多篇相关内容的合并解读 |
| 文集型 | 多作者多观点的话题合集，呈现立场谱系 |

支持三种输出模式：

| 模式 | 用户真正想完成的事 | 适合场景 | 输出长度 |
|---|---|---|---:|
| Quick 决策版 | 判断这份内容值不值得深入 | 想先看核心、再决定要不要继续 | 1500-2500 字 |
| Standard 学习版 | 真正学到这份内容最有用的核心 | 想得到一篇完整、好读、有深度的解读 | 3500-5000 字 |
| Deep 重构版 | 把复杂内容重建成一套完整理解系统 | 适合机制复杂、方法密集，或要做长文/课程底稿 | 6000-8000 字 |

请回复：

1. 你要处理的素材：上传文件、粘贴正文、提供书名或链接均可；
2. 选择模式（可不选，我会自动推荐）；
3. 目标读者（可不填，默认"一二线城市普通中文读者"）；
4. 用途（可不填，默认读者视角，跳过骨架直接出成稿；如要发布请说明）。
```
